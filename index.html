  • Fork a repo.
  • Add your name to the readme.
  • Open pull request.

    • Things to understand

  • Repositories: A repository is like a project's folder, which contains all the files, folders, and the entire history of changes for your project.
  • Commits: A commit represents a specific version of your project. Each commit captures a snapshot of your code at a particular point in time.
  • Branches: Branches are parallel versions of your code. They allow multiple developers to work on different features or bug fixes simultaneously.
  • Merge: Merging combines changes from one branch into another. It helps consolidate work and resolve conflicts that may arise when different branches modify the same file

    Commands Learnt

  • git init: initialize a git repository.
  • git add: Add your project files to git
  • git commit: Add a meaningful message about the changes made
  • git push: uploads the committed changes (or files in the staging area) to a repository
